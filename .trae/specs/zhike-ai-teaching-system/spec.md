# 基于多模态大模型的智能教学闭环系统 - 产品需求文档

## Overview
- **Summary**: 基于多模态大模型的智能教学闭环系统，采用前后端分离架构，集成AI能力实现智能问答、视频解析、学情分析等功能。
- **Purpose**: 解决传统教学中知识传递效率低、个性化学习不足、学情分析困难等问题，通过AI技术提升教学效果和学习体验。
- **Target Users**: 学生、教师、系统管理员。

## Goals
- 实现基于多模态大模型的智能问答系统，支持视频内容的理解和交互
- 构建视频解析系统，自动提取知识片段并生成结构化内容
- 开发学情分析模块，提供可视化的学习数据分析和热力图
- 实现多角色权限管理，支持学生、教师、管理员的不同操作权限
- 提供沉浸式学习体验，支持双主题切换和响应式设计

## Non-Goals (Out of Scope)
- 实时视频直播功能
- 在线考试系统
- 第三方支付集成
- 多语言国际化支持
- 移动端原生应用开发

## Background & Context
- 本项目是一个毕业设计重构项目，旨在利用现代技术栈和AI能力提升教学系统的智能化水平
- 技术栈包括Vue 3 + TypeScript + Spring Boot 3 + Java 17 + MySQL 8 + Redis + Elasticsearch
- 接入阿里云通义千问 (Qwen) 等大模型提供AI能力
- 采用模块化单体架构，前后端分离设计

## Functional Requirements
- **FR-1**: 用户认证与授权系统，支持JWT + RBAC权限管理
- **FR-2**: 视频上传与解析系统，支持FFmpeg抽帧和AI分析
- **FR-3**: RAG智能问答系统，支持流式响应和知识溯源
- **FR-4**: 学情分析系统，生成热力图和学习画像
- **FR-5**: 教师管理系统，支持课程和视频管理
- **FR-6**: 学生学习系统，支持课程浏览、视频观看和笔记管理
- **FR-7**: 管理员系统，支持用户管理和系统配置
- **FR-8**: 多角色主题系统，支持学生端双主题和教师/管理端浅色主题

## Non-Functional Requirements
- **NFR-1**: 性能要求：视频解析响应时间不超过30秒，智能问答响应时间不超过2秒
- **NFR-2**: 安全性：采用BCrypt加密存储密码，JWT token认证，防止SQL注入和XSS攻击
- **NFR-3**: 可扩展性：模块化设计，支持后续功能扩展和模型升级
- **NFR-4**: 可用性：系统正常运行时间达到99.9%，支持容错处理
- **NFR-5**: 响应式设计：适配桌面端、平板和移动设备

## Constraints
- **Technical**: 前端使用Vue 3 + TypeScript + Vite，后端使用Spring Boot 3 + Java 17，数据库使用MySQL 8 + Redis + Elasticsearch
- **Business**: 作为毕业设计项目，需要在规定时间内完成核心功能开发
- **Dependencies**: 依赖阿里云DashScope SDK、FFmpeg、RabbitMQ等外部服务

## Assumptions
- 系统部署环境具备足够的计算资源支持视频处理和AI模型推理
- 阿里云服务账号已配置并具备相应的API调用权限
- 用户具备基本的网络环境和设备条件使用系统

## Acceptance Criteria

### AC-1: 用户认证系统
- **Given**: 用户输入正确的用户名和密码
- **When**: 用户点击登录按钮
- **Then**: 系统验证凭证并返回JWT token，用户成功登录
- **Verification**: `programmatic`

### AC-2: 视频上传与解析
- **Given**: 教师用户上传视频文件
- **When**: 系统接收到视频文件并触发解析
- **Then**: 系统完成视频解析并生成知识片段
- **Verification**: `programmatic`

### AC-3: RAG智能问答
- **Given**: 学生用户在学习过程中提出问题
- **When**: 系统接收到问题并进行RAG处理
- **Then**: 系统返回基于视频内容的智能回答，并提供知识溯源
- **Verification**: `programmatic`

### AC-4: 学情分析
- **Given**: 教师用户查看班级学情
- **When**: 系统生成学情热力图和学习数据
- **Then**: 教师可以清晰看到学生学习情况和难点分布
- **Verification**: `human-judgment`

### AC-5: 多角色主题系统
- **Given**: 不同角色用户登录系统
- **When**: 系统根据角色自动加载对应主题
- **Then**: 学生端显示双主题界面，教师/管理端显示浅色主题
- **Verification**: `human-judgment`

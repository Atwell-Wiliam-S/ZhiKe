# 学生组接口实现 - 产品需求文档

## 概述
- **摘要**：实现智能教学闭环系统的学生组接口，包括课程列表、课程详情、视频播放信息、心跳上报、进度保存和笔记管理等功能。
- **目的**：为学生用户提供完整的学习体验，支持课程浏览、视频学习、进度追踪和笔记管理等核心功能。
- **目标用户**：系统中的学生角色用户。

## 目标
- 实现学生组接口的完整功能，包括课程列表、课程详情、视频播放信息、心跳上报、进度保存和笔记管理。
- 确保接口符合 API 规范，包括统一响应格式、错误处理和权限控制。
- 提供清晰的实现计划，支持分步执行和测试。

## 非目标（范围外）
- 不包括教师和管理员接口的实现。
- 不包括前端实现和 UI 设计。
- 不包括数据库 schema 设计和初始化。

## 背景与上下文
- 系统采用前后端分离架构，后端使用 Spring Boot 3.2.0，前端使用 Vue 3 + TypeScript。
- 认证采用 JWT + RBAC 模式，确保接口安全。
- 数据库使用 MySQL 8.0.35，存储课程、视频、学习记录等数据。
- 接口规范已在 `03-API接口规范.md` 中定义，需要严格按照规范实现。

## 功能需求
- **FR-1**：实现 `/api/student/courses` 接口，支持课程列表查询，包含关键字搜索和分页。
- **FR-2**：实现 `/api/student/courses/:id` 接口，支持课程详情查询，包含视频列表。
- **FR-3**：实现 `/api/student/videos/:id/play-info` 接口，支持视频播放信息查询，包含签名 URL 和知识片段。
- **FR-4**：实现 `/api/student/records/heartbeat` 接口，支持学习心跳上报，每 10 秒更新学习记录。
- **FR-5**：实现 `/api/student/records/progress` 接口，支持学习进度保存。
- **FR-6**：实现 `/api/student/notes` 接口，支持添加笔记和查询笔记列表。

## 非功能需求
- **NFR-1**：接口响应时间不超过 500ms，确保良好的用户体验。
- **NFR-2**：支持并发请求，能够处理至少 100 个并发用户。
- **NFR-3**：严格遵循 API 规范，包括统一响应格式、错误处理和权限控制。
- **NFR-4**：代码结构清晰，遵循 Spring Boot 最佳实践，便于维护和扩展。

## 约束
- **技术**：使用 Spring Boot 3.2.0，MyBatis-Plus 3.5.5，MySQL 8.0.35。
- **业务**：严格按照 API 规范实现，确保与前端对接顺畅。
- **依赖**：依赖现有的认证系统和数据库结构。

## 假设
- 数据库表结构已按照设计文档创建完成。
- 认证系统已实现，能够正确处理 JWT Token。
- OSS 签名 URL 生成功能已实现。

## 验收标准

### AC-1：课程列表接口
- **给定**：学生用户已登录，拥有有效的 JWT Token。
- **当**：发送 GET 请求到 `/api/student/courses`，携带查询参数 `keyword`、`page` 和 `pageSize`。
- **然后**：返回符合条件的课程列表，包含分页信息，响应格式符合规范。
- **验证**：`programmatic`

### AC-2：课程详情接口
- **给定**：学生用户已登录，拥有有效的 JWT Token。
- **当**：发送 GET 请求到 `/api/student/courses/:id`，其中 `id` 是有效的课程 ID。
- **然后**：返回课程详情，包含视频列表，响应格式符合规范。
- **验证**：`programmatic`

### AC-3：视频播放信息接口
- **给定**：学生用户已登录，拥有有效的 JWT Token。
- **当**：发送 GET 请求到 `/api/student/videos/:id/play-info`，其中 `id` 是有效的视频 ID。
- **然后**：返回视频播放信息，包含签名 URL 和知识片段，响应格式符合规范。
- **验证**：`programmatic`

### AC-4：心跳上报接口
- **给定**：学生用户已登录，拥有有效的 JWT Token。
- **当**：发送 POST 请求到 `/api/student/records/heartbeat`，携带视频 ID、当前时间和播放状态。
- **然后**：后端更新学习记录，返回成功响应，响应格式符合规范。
- **验证**：`programmatic`

### AC-5：进度保存接口
- **给定**：学生用户已登录，拥有有效的 JWT Token。
- **当**：发送 POST 请求到 `/api/student/records/progress`，携带视频 ID、当前时间和总时长。
- **然后**：后端更新学习进度，返回成功响应，响应格式符合规范。
- **验证**：`programmatic`

### AC-6：笔记管理接口
- **给定**：学生用户已登录，拥有有效的 JWT Token。
- **当**：发送 POST 请求到 `/api/student/notes`，携带视频 ID、时间戳和笔记内容。
- **然后**：后端创建笔记，返回笔记 ID，响应格式符合规范。
- **当**：发送 GET 请求到 `/api/student/notes`，携带视频 ID。
- **然后**：返回该视频的笔记列表，响应格式符合规范。
- **验证**：`programmatic`
